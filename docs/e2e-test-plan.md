## 1. 测试架构与基础设施

### 1.1 测试文件结构

```
apps/web/e2e/
├── helpers/
│   ├── vault.ts          # createVault, unlockVault, lockVault 通用函数
│   ├── credentials.ts    # addCredential, editCredential, deleteCredential
│   ├── auth.ts           # registerAccount, loginAccount, logoutAccount
│   └── constants.ts      # 测试密码、邮箱等常量
├── vault-flow.spec.ts        # 模块 A~E（本地功能）
├── import-flow.spec.ts       # 模块 F（导入）
├── generator.spec.ts         # 模块 G（密码生成器）
├── dashboard.spec.ts         # 模块 H（仪表盘）
├── settings.spec.ts          # 模块 I, M（设置与导出）
├── worker-sync.spec.ts       # 模块 J（同步联调）
├── device-trust.spec.ts      # 模块 K（设备信任联调）
├── recovery.spec.ts          # 模块 L（恢复备份）
├── cloud-export.spec.ts      # 模块 N（云端备份联调）
├── totp.spec.ts              # 模块 O（TOTP）
├── offline.spec.ts           # 模块 Q（离线）
├── security.spec.ts          # 模块 T（安全边界）
└── api-integration.spec.ts   # 模块 U（API 端点）
```

### 1.2 Playwright 配置要求

```typescript
// playwright.config.ts 需要支持多个 spec 文件
export default defineConfig({
  testDir: "./e2e",
  testMatch: /\.spec\.ts/u,  // 匹配所有 spec 文件
  timeout: 120_000,          // 联调测试需要更长超时
  retries: 0,
  use: {
    baseURL: "http://localhost:3001",
    screenshot: "only-on-failure",
    trace: "retain-on-failure",
    viewport: { width: 1440, height: 900 },
  },
  projects: [
    { name: "chromium", use: { browserName: "chromium" } },
  ],
  webServer: [
    {
      command: "pnpm exec next dev --port 3001",
      port: 3001,
      reuseExistingServer: true,
      timeout: 120_000,
    },
    {
      command: "pnpm dev:worker",
      port: 8787,
      reuseExistingServer: true,
      timeout: 120_000,
    },
  ],
});
```

### 1.3 通用测试工具函数

| 函数 | 用途 |
|---|---|
| `createVault(page, password?)` | 创建新密码库并等待解锁 |
| `unlockVault(page, password)` | 解锁已有密码库 |
| `lockVault(page)` | 锁定密码库 |
| `addCredential(page, opts)` | 新增凭据（login / secure_note / credit_card） |
| `editCredential(page, title, changes)` | 编辑指定凭据 |
| `deleteCredential(page, title)` | 删除指定凭据 |
| `registerAccount(page, email, password)` | 注册云端账户 |
| `loginAccount(page, email, password)` | 登录云端账户 |
| `navigateTo(page, navId)` | 切换侧边栏导航 |
| `generateUniqueEmail()` | 生成唯一测试邮箱 |

## 2. 模块 A：密码库创建与解锁

### A-01: 首次创建密码库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 访问 `/` | 显示创建主密码界面（FORGE MODE），标题包含"创建你的主密码" |
| 2 | 输入主密码 `TestPassword123!Secure` | 密码输入框 `#master-password` 显示已填入 |
| 3 | 点击"创建密码库"按钮 | 按钮变为 loading 状态 |
| 4 | 等待解锁完成 | 自动跳转至 `/vault`，显示 `.stats-grid` 仪表盘 |
| 5 | 检查侧边栏 | "密码列表"、"导入密码"、"设备同步"等导航项可用 |
| 6 | 检查主密码输入框 | `#master-password` 不再可见 |

### A-02: 主密码为空时创建

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 访问 `/`，不输入密码 | 创建按钮禁用或提交后显示错误 |
| 2 | 提交空表单 | 显示表单验证错误 |

### A-03: 解锁已有密码库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 创建密码库后锁定 | 返回 `/`，显示 UNLOCK MODE（解锁模式） |
| 2 | 输入正确主密码 | 成功解锁，跳转 `/vault`，数据完整 |
| 3 | 输入错误主密码 | 显示"主密码不正确，或本地密码库已损坏。" |

### A-04: 锁定密码库

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 解锁后点击"锁定密码库" | 返回 `/`，所有凭据数据不可见 |
| 2 | 检查状态 | 搜索框清空，抽屉关闭，筛选重置 |
| 3 | 检查导航 | 默认回到"密码总览" |

### A-05: 自动锁定

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 设置自动锁定为 1 分钟 | 设置页选择"1 分钟" |
| 2 | 解锁密码库后不做任何操作 | 等待 60 秒 |
| 3 | 观察状态 | 自动锁定，返回锁屏界面，状态显示"自动锁定（1分钟无操作）" |

### A-06: 自动锁定计时器重置

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 解锁密码库 | 倒计时开始 |
| 2 | 在即将锁定前移动鼠标 | 计时器重置，不会锁定 |
| 3 | 验证键盘操作也能重置 | 按任意键，计时器重置 |

### A-07: 路由守卫

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 锁定状态直接访问 `/vault` | 自动重定向到 `/` |
| 2 | 解锁状态直接访问 `/` | 自动重定向到 `/vault` |
