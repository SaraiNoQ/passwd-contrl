## 验证命令

运行以下命令验证基础质量：

```sh
# TypeScript 类型检查
npx pnpm typecheck

# 单元测试
npx pnpm test

# 全量构建
npx pnpm build

# 扩展 E2E 测试
npx pnpm --filter @zero-vault/extension test:e2e

# Rust 测试
cargo test --manifest-path crates/crypto-core/Cargo.toml

# WASM 构建
npx pnpm wasm:build
```

安全泄漏回归测试：

```sh
npx pnpm --filter @zero-vault/web test -- --grep "leakage"
npx pnpm --filter @zero-vault/worker-api test -- --grep "leakage"
```

## 手动验证流程

以下项目需要在真实浏览器中手动验证，无法通过自动化测试覆盖：

1. **Chrome 手动 E2E**：加载 unpacked extension，在真实网站上测试填充、阻止、popup 行为。
2. **Edge 手动 E2E**：在 Microsoft Edge 上重复 Chrome 手动测试。
3. **跨浏览器填充兼容性**：确认 content script 在 Chrome 和 Edge 中的行为一致。
4. **恢复码端到端流程**：生成码 → 离线保存 → 模拟恢复 → 验证 vault 可解锁。
5. **设备信任端到端流程**：注册设备 → 从已有设备审批 → 撤销设备 → 验证撤销生效。
6. **中文文本溢出检查**：在扩展 popup（320px）和移动视口下检查所有中文文案不溢出。
7. **深色主题对比度检查**：在真实显示器上验证文本可读性，特别是 `--color-text-muted` (#94A3B8) 在 `--color-bg-panel` (#0B1624) 上的对比度。
